# Publication-Lifecycle Terminology Remediation

## Goal

Exhaustively audit and correct publication-lifecycle terminology that has leaked into reusable runtime architecture. Ensure terms like "journal", "conference", "paper", "manuscript", "publication", "camera_ready", "extension" do not name runtime behavior, configuration, artifacts, package structures, executable protocols, methods, variables, classes, dataclasses, enums, or file names — unless they are legitimate research/documentation/reporting terminology.

## Governing Principle

Publication destination must not become the name of reusable software behavior.
