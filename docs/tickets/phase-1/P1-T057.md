13. **Objective.** Implement `infrastructure/persistence/artifacts.py`: content-addressed lookup, and atomic single-file commit — a temporary file written in the destination directory or the same validated filesystem, a complete write, flush/fsync per policy, content-hash and schema verification, an atomic replace, and a manifest update only after the replace has succeeded.
14. **Why this ticket exists.** This ticket was split out of a former combined persistence mega-ticket because single-file atomic commit has its own correctness protocol (Architecture §15.3), structurally distinct from multi-file bundle commit (P1-T058), which requires a different marker-based visibility protocol. The exact ordering (write, verify, replace, then update the manifest — never the reverse) is what prevents a partially-written file from ever being cited as complete: if the manifest were updated before the replace succeeded, a crash between those two steps would leave a manifest pointing at content that does not yet exist on disk.
15. **Authority references.** Architecture §12.2 (`ArtifactStore` port, single-file methods), §12.4 (`ArtifactStore` implementation), §15.3 (single-file writer protocol), §15.4 (storage resolution diagram), §29.4 (invalid states 33–34: a partial file is never marked complete; an immutable artifact is never silently overwritten).
16. **Dependencies.** P1-T054, P1-T055, P1-T056.
17. **Blocks.** P1-T058, P1-T059, P1-T070.
18. **Downstream consumers.** Every stage that persists a single-file scientific artifact — source manifests, split manifests, threshold outputs, metric outputs, statistical outputs, resolved-configuration artifacts — depends on this ticket transitively through `ArtifactStore`.
19. **Preconditions.** P1-T054, P1-T055, P1-T056 are `DONE`.
20. **Inputs.** Architecture §12.2, §12.4, §15.3, §15.4, §29.4.
21. **Allowed scope.** `infrastructure/persistence/artifacts.py`.
22. **Forbidden actions.** No claim of directory-level atomicity derived from a single `os.replace` call; no manifest update before the file has been verified and replaced; no overwrite of a hash-verified immutable artifact with different content under the same logical id (an identical logical id with mismatched persisted bytes is always a typed integrity conflict, never silently resolved by minting a new random id).
23. **Required configuration or NONE.** The write-disposition policy (`CREATE_IF_ABSENT`, `VERIFY_OR_FAIL`, `ATOMIC_STAGE_COMMIT`) comes from `ArtifactPolicy`.
24. **Implementation responsibilities.** Implement content-addressed `lookup`; implement `write_atomically` following the exact same-filesystem temp-write/verify/replace/manifest-update sequence; implement `validate_integrity` re-verifying the hash and schema against the persisted manifest.
25. **Required unit tests.** A write that fails partway through leaves no partial artifact ever marked complete; an identical logical id with mismatched persisted bytes raises a typed integrity error rather than being silently overwritten or re-keyed under a new id.
26. **Required property tests.** Repeated `write_atomically` calls with identical content are idempotent — deduplicated, never re-written or re-verified redundantly.
27. **Required contract tests.** `ArtifactStore`'s single-file methods satisfy the Section 12.2 port contract exactly, with no extra method, no missing method, and no weakened generic signature.
28. **Required architecture tests.** No framework carrier object (a NumPy array, a pandas frame, a PyArrow table, a Torch tensor) crosses the `ArtifactStore` port signature.
29. **Required integration tests.** A same-filesystem atomic commit on synthetic content; a simulated killed-mid-write scenario leaves no partial artifact and no corrupted manifest.
30. **Required CUDA tests.** NONE.
31. **Required system/end-to-end tests.** NONE.
32. **Validation commands.** Unit lane; property lane; contract lane; architecture lane; integration lane (persistence).
33. **Acceptance criteria.** [ ] The commit is atomic on a single filesystem. [ ] No partial artifact is ever marked complete. [ ] A mismatched-bytes-same-id conflict is always a typed integrity error. [ ] The manifest updates only after the replace succeeds.
34. **Completion evidence.** Changed files: `artifacts.py` and its tests. Commands run and results. Cleanup confirmation. Confirmation of no unrelated modification.
35. **Failure and blocker behavior.** `ArtifactError`/`PartialArtifactError` is raised on any incomplete or corrupt write; the affected stage recomputes rather than reusing a partial artifact.
