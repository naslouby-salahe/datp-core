13. **Objective.** Implement `infrastructure/persistence/bundles.py`: immutable multi-file bundle commit — write and verify every declared member (schema, row count/order, lineage, content hash), then write the final commit marker last, so a reader always rejects an absent or invalid marker or any member-hash mismatch and never observes a partially-written bundle.
14. **Why this ticket exists.** This ticket was split out because the multi-file bundle protocol (marker-based visibility) is structurally different from single-file atomic replace (P1-T057), and it is the exact mechanism the atomic per-client test-score aggregate (benign plus attack scores, built in P1-T023) depends on: `ClientTestScoreArtifact` is defined as one committed aggregate specifically because this ticket makes "one aggregate" a real, enforced storage guarantee rather than a type-level promise with no backing implementation. Writing the commit marker last — after every member has been individually verified — is what makes an interrupted bundle commit invisible to readers rather than silently half-present.
15. **Authority references.** Architecture §9.2 (atomic test-score-pair persistence protocol, steps 1–9), §12.2 (`ArtifactStore.commit_bundle`), §15.3 (multi-file writer protocol), §29.4 (invalid state 33: a partial bundle is never marked complete).
16. **Dependencies.** P1-T057.
17. **Blocks.** P1-T070.
18. **Downstream consumers.** Every scoring/evaluation stage that commits a `ClientTestScoreArtifact` aggregate or any other fully-typed multi-file bundle depends on this ticket directly.
19. **Preconditions.** P1-T057 is `DONE`.
20. **Inputs.** Architecture §9.2, §12.2, §15.3.
21. **Allowed scope.** `infrastructure/persistence/bundles.py`.
22. **Forbidden actions.** No claim of directory-level atomicity derived from individual `os.replace` calls on the bundle's member files; no reader-visible bundle before its commit marker is written and verified; no loosely-typed or partially-committed pair ever accepted as a substitute for a fully declared bundle.
23. **Required configuration or NONE.** NONE.
24. **Implementation responsibilities.** Implement `commit_bundle`: create a new immutable bundle directory, write and verify every declared member, compute and record member hashes in `ArtifactBundleManifest`, write the final commit marker last, and return `ArtifactBundleCommitResult` only after the marker itself has been verified.
25. **Required unit tests.** A missing benign or attack child artifact is rejected; a mismatched client, split, checkpoint, or scoring identity between declared members is rejected; an absent or invalid final commit marker makes the entire bundle unreadable; a child content-hash mismatch is rejected.
26. **Required property tests.** NONE.
27. **Required contract tests.** `ArtifactStore.commit_bundle` satisfies its Section 12.2 signature exactly; a successfully committed bundle is readable and always consumed as one aggregate, never as two independent references.
28. **Required architecture tests.** No framework carrier object crosses the `commit_bundle` signature.
29. **Required integration tests.** A synthetic two-member bundle (benign plus attack score children) commits atomically and is correctly rejected as unreadable if interrupted before the marker is written.
30. **Required CUDA tests.** NONE.
31. **Required system/end-to-end tests.** NONE.
32. **Validation commands.** Unit lane; contract lane; architecture lane; integration lane (persistence).
33. **Acceptance criteria.** [ ] The commit marker is always written last, with no reader-visible partial bundle. [ ] Any missing or mismatched member is rejected. [ ] A committed pair is always consumed as exactly one aggregate. [ ] No directory-atomicity overclaim exists anywhere.
34. **Completion evidence.** Changed files: `bundles.py` and its tests. Commands run and results. Cleanup confirmation. Confirmation of no unrelated modification.
35. **Failure and blocker behavior.** `IncompleteArtifactBundleError`/`PartialArtifactError` is raised on any missing, mismatched, or uncommitted member; the bundle is recomputed in full, never partially exposed.
36. **Stop conditions.** None.
