14. **Why this ticket exists.** Architecture §8.1 states that the scientific meaning of an experiment lives entirely inside `ScientificProtocolSpec`, with execution/artifact/reporting concerns held in three deliberately separate policy aggregates — this is the structural prevention of a "universal context object" (Architecture principle 10) that would otherwise let an operational detail (a machine path, a rendering format) accidentally enter a scientific fingerprint. Each field of `ScientificProtocolSpec` must contribute only to the earliest stage identity it can affect: a threshold percentile changes `ThresholdIdentity` and its downstream identities only; a partition seed changes `PartitionIdentity` and everything downstream of it; a reporting format changes nothing scientific at all. Getting this composition wrong would silently over- or under-invalidate cached artifacts throughout the reuse/lineage system built in later tickets (P1-T033, P1-T038).
