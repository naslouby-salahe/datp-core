14. **Why this ticket exists.** Scoring must stream batches to device with incremental writes because a whole-client or whole-dataset scoring pass loaded into GPU memory at once would defeat Architecture principle 12's memory-safety guarantee exactly as whole-dataset loading would for data adapters (P1-T042). Canonical B4 must use exact `KMeans`, never `MiniBatchKMeans`, because approximately nine to fifteen four-scalar client fingerprints (Architecture §16.2) do not remotely justify the approximation error a mini-batch algorithm would introduce for such a small input — this is a concrete instance of Architecture principle 11 ("selective abstraction... a port exists only at a genuine variation point"), applied here as "an approximation exists only where the input size actually demands it." The FedStats algorithm's structural rules (full pooled variance, larger-k tie-break) cannot be disabled by any caller-supplied flag, closing off exactly the kind of post-hoc tuning the roadmap's SB-22 forbids ("do not tune the `B-FedStatsBenign` protocol after seeing results").
