14. **Why this ticket exists.** Roadmap §10 locks the journal checkpoint protocol: train once to a maximum of 200 rounds, save and evaluate at `{25, 50, 75, 100, 125, 150, 200}`, and Regime A alone selects one global primary checkpoint used for every main-regime table — per-regime selection, test-AUROC selection, attack-label selection, and hiding weak checkpoint curves are all forbidden. Architecture §17.1 states the candidate request type cannot carry test AUROC, attack labels, Regime D, FedProx, personalization, poisoned data, or downstream threshold results as fields, because if any of those could reach the selection evidence, a later stress-test or external-validation result could retroactively bias which round is declared the main-regime checkpoint. Architecture §17.2 keeps `RecoveryState` structurally distinct so an operational resume point can never be cited as, or substituted for, the scientifically selected round.

Roadmap §10 (locked journal checkpoint protocol: train-once/save-many, Regime-A-global selection, forbidden per-regime/test-AUROC/attack-label selection).
