13. **Objective.** Implement `infrastructure/persistence/roots.py` and `paths.py`: bind each `StorageRootSpec` to a concrete `BoundStorageRoot`, and implement the internal `ArtifactPathResolver.resolve(ResolveArtifactLocationRequest)`, which exhaustively matches the closed `ArtifactKey` union with `assert_never`, derives a `RelativeArtifactPath`, and joins it beneath the bound root — raising `PathResolutionError` on any attempted escape.
14. **Why this ticket exists.** This ticket was split out of a former combined path/hashing/serialization/persistence/lock mega-ticket because path resolution is a distinct, narrowly-scoped internal collaborator (Architecture §15) that must never be exposed as an application-facing port and must never be conflated with hashing (P1-T055), serialization (P1-T056), atomic commit (P1-T057), bundle commit (P1-T058), or locking (P1-T059) — each of those has its own distinct failure mode and its own dedicated ticket. Keeping `ArtifactPathResolver` strictly internal to `infrastructure` matters because `application` code must reason about semantic storage roots and typed artifact keys, never concrete filesystem paths (Architecture §15.1); exposing this resolver as a port would let a path leak into a place where identity should be computed instead.

24. **Implementation responsibilities.** Bind every `StorageRootSpec` to a concrete `BoundStorageRoot`; implement content-addressed sharded relative-location derivation (`<hash[:2]>/<hash>`); implement `ArtifactPathResolver.resolve` with exhaustive `assert_never` matching over the six `ArtifactKey` variants; reject any resolution that would escape the bound root.
25. **Required unit tests.** A resolution attempt that would escape its `BoundStorageRoot` (via a crafted key or traversal-like input) raises `PathResolutionError`; each of the six `ArtifactKey` variants resolves to a distinct, stable relative location; adding a hypothetical seventh `ArtifactKey` variant without a corresponding resolver branch is caught statically by `assert_never`, not merely at runtime.
26. **Required property tests.** Identical logical inputs always resolve to the identical relative location across repeated calls; content-addressed sharding is stable under repeated resolution of the same content hash.
27. **Required contract tests.** `ArtifactPathResolver` is never invoked directly by `application` — only `infrastructure/persistence` adapters call it (a contract-level import/usage check, not merely a docstring claim).
28. **Required architecture tests.** No `Path` construction exists anywhere outside `infrastructure/persistence`; `domain` and `application` contracts reference only `StorageRootKind`/`StorageRootSpec`, never a concrete path type.
29. **Required integration tests.** A synthetic multi-root resolution round-trip exercising dataset/regime/seed-scoped/cross-seed/run/report keys produces the expected relative locations beneath each bound root.
30. **Required CUDA tests.** NONE.
31. **Required system/end-to-end tests.** NONE.
