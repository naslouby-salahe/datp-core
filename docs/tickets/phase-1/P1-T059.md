14. **Why this ticket exists.** This ticket was split out because locking has a distinct failure mode — contention, staleness, and process death — and a distinct scope model that must not be conflated with the artifact/bundle commit protocols themselves (P1-T057/P1-T058): `COMPUTATION_OWNERSHIP` may be held for the duration of a long training run, while `COMMIT` is always short-lived, held only around final persistence. The master log's Section B explicitly states a filesystem commit lock is never held across a long training run — this ticket is the concrete implementation of that separation, so a computation-ownership lease can survive an hours-long federated training round while a commit lease is released within moments of a successful write.
