### B2. Anchor-campaign scientific locks

Phase 3 is the DATP anchor campaign only. The Phase 3 campaign is neither a single process, command, job, physical write, nor attempt. A scientific configuration determines one `RunIdentity`; a changed scientific configuration is a new campaign. Each operational attempt has a distinct `ExecutionAttemptId` and never silently changes the campaign identity.

Keep resume, compatible recovery, bounded transient-infrastructure retry, diagnosed technical-invalidity correction, affected-stage recomputation, and new attempt versus new campaign distinct. Only diagnosed technical invalidity may justify corrective execution, and only with preserved failed-attempt evidence, root cause, regression tests, unchanged scientific configuration, affected-stage scope, explicit approval, and a new attempt identity where applicable. Never use a preferred result direction to justify retry, seed selection, threshold/checkpoint/eligibility/preprocessing changes, suppression, or replacement.

The anchor ladder preserves fixed FedAvg encoder/model state and compatible benign-calibration and test-score artifacts across B1–B4; threshold scope alone varies. B0 is a separate centralized lineage. B3 requires its authorized family semantics. B4 uses the four-scalar `[mean, std, skew, p95]` fingerprint at canonical `K=3`. Calibration is benign-only; attack data are evaluation-only. CV(FPR) is the operating-point endpoint and AUROC is a control. The anchor cohort and honesty diagnostic are not the journal’s ten-seed confirmatory verdict.
