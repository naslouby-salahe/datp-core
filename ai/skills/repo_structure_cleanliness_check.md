# Repo Structure Cleanliness Check

## Purpose
Prevent repository clutter and unclear ownership.

## When to apply
Apply after every file-changing task.

## Blocking rules
Block ugly folders, duplicated modules, redirects, shims, fake compatibility, temp files, audit clutter, scratch files, random root files, stale docs, unclear ownership, and hidden tool-specific folders.

## Pass criteria
Only approved files changed and the repository structure remains direct.

## Fail criteria
Temp, audit, report, scratch, migration, compat, legacy, or hidden agent clutter is created.
