# DATP Journal Scope Guard

## Purpose
Keep work inside the DATP journal-extension scope.

## When to apply
Apply to every task that touches protocol, experiments, results, claims, or repository governance.

## Blocking rules
Block poisoning, Dynamic DATP, privacy guarantees, deployment profiling, backdoor, evasion, full drift detection, generic FL-IDS expansion, release work, tag work, and versioning work unless explicitly requested as future-work wording.

## Pass criteria
The task preserves fixed encoder, fixed federated model, threshold-calibration-scope framing, B1-B4 threshold-scope variants, and unchanged trained model and score artifacts.

## Fail criteria
Scope expands, stress tests replace the causal ladder, or claims imply unsupported scientific meaning.
