# Naming Clarity Check

## Purpose
Keep names clear, current, and semantically accurate.

## When to apply
Apply after any edit to files, modules, functions, classes, variables, configs, outputs, experiments, or docs.

## Blocking rules
Block weird names, vague names, stale names, misleading names, overloaded names, old aliases, and compatibility names.

## Pass criteria
Names are direct, readable, current, and map to one concept.

## Fail criteria
Old and new names coexist, names hide meaning, or abbreviations create ambiguity.
