# Test Quality Check

## Purpose
Keep tests useful, focused, and tied to behavior.

## When to apply
Apply when tests are added, edited, selected, or skipped.

## Blocking rules
Block bloated tests, trivial tests, duplicate tests, implementation-detail tests, low-value tests, failing impacted tests, and skipped impacted tests without reason.

## Pass criteria
Tests cover behavior, metrics, configs, artifacts, regressions, or documented workflow risk.

## Fail criteria
Tests add noise, miss the impacted behavior, or are not run after code changes.
