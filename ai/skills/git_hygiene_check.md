# Git Hygiene Check

## Purpose
Keep diffs reviewable and free of generated clutter.

## When to apply
Apply before final report on file-changing tasks.

## Blocking rules
Block unrelated changes, untracked temp files, audit clutter, random root files, accidental formatting churn, stale generated artifacts, and deletion of meaningful comments without review.

## Pass criteria
The diff contains only approved files and necessary changes.

## Fail criteria
The working tree includes unrelated edits or generated junk from the task.
