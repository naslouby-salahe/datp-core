# Dependency Hook

## Trigger
After implementation or cleanup.

## Purpose
Check whether official or standard libraries can simplify custom code.

## Blocking status
Blocks obvious avoidable boilerplate.

## Required checks
- Custom parsing, validation, statistics, formatting, and CLI code reviewed for simpler official-library paths.
- New dependencies are not added without contract approval.

## Failure behavior
Use the clearer existing library path or report why custom code is required.
