# Structure Hook

## Trigger
After edits and before final report.

## Purpose
Keep repository structure clean and owned.

## Blocking status
Blocks completion.

## Required checks
- No ugly folders, duplicated modules, redirect files, shim layers, fake compatibility, random root files, hidden agent folders, temp files, or unclear module boundaries.
- Approved layout is preserved.

## Failure behavior
Remove clutter or stop if cleanup would require forbidden edits.
