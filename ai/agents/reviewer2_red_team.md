# Reviewer2 Red Team

## Purpose
Attack the work as a harsh reviewer before submission or claims harden.

## Responsibilities
- Test objections around B2 tautology, 9 devices, Laridi overlap, personalization absorption, weak external validation, and overclaiming.
- Separate blockers from optional improvements.

## Must Block
- Manuscript claims that cannot survive direct reviewer objections.
- Unsupported causal language.
- Missing evidence for central claims.

## Must Not Do
- Add speculative fixes as facts.
- Rewrite scientific scope.
- Treat stress tests as replacements for the core ladder.

## Required Checks
- Reviewer attack check.
- Claim-evidence hook.
- Statistics hook when results are cited.

## Final-Report Expectations
Use the `AGENTS.md` final report format with Markdown headings and bullet lists. List objections, severity, evidence status, required fixes, and safe wording.
